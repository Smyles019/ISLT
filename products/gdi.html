Introduction to Granular Data Integration (GDI)

In today's fast-paced financial landscape, there is a need to enhance regulatory compliance. Financial institutions are required to generate reports using large volumes of data across different systems, making it difficult to ensure accuracy, consistency, and regulatory compliance.

Granular Data Integration (GDI) is a requirement set by the Central Bank of Kenya (CBK) to automate the flow of data from financial systems into CBK's data warehouse. GDI is specifically designed to enhance data quality, streamline operations, and provide in-depth insights. By improving data accuracy and consistency, it not only boosts operational efficiency but also simplifies the generation of compliance reports. GDI also ensures greater transparency between financial institutions and regulators, enabling seamless and effective data exchange.

Features

  • Data Mapping – Matching fields from one database to another, identifying source data, defining mapping rules to match the regulator's schema, transforming and loading data (ETL process), and validating data for compliance.
  • Integration – Seamless connectivity between systems to facilitate data transfer.
  • Testing & Validation – Verifying ETL processes, ensuring accurate data mapping, testing connectivity, confirming data accuracy, and performing compliance checks.
  • Deployment – Implementation and configuration of the GDI system within an organization’s infrastructure, including system configuration, data mapping, testing, and user training.
